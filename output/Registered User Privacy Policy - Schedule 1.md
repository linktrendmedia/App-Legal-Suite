## 7. DATA RETENTION PERIODS

### 7.1 Account Data

**Retention:** Until account deletion

### 7.2 Transaction Data

**Retention:** 7 years (for tax and accounting requirements)

### 7.3 Support Records

**Retention:** 12 months after closure of the support ticket

### 7.4 Logs and Security Data

**Retention:** 12 months

### 7.5 Backups

**Retention:** 90 days (backup rotation schedule)

**Backup Deletion:** Residual copies in backups are securely overwritten during normal backup cycles.

### 7.6 Marketing Data

**Retention:** Until consent is withdrawn or deletion is requested

### 7.7 Deletion Timeline

**30 days** from receipt of valid deletion request (aligns with DPA 13.2, MSA 21.4, EULA 9.4(b), B2C 13.4(b), Website 12.6(b), Public Privacy 12.2(b))

## 8. SECURITY MEASURES

### 8.1 Encryption

**(a)** In transit: TLS 1.2+ (TLS 1.3 preferred)  
**(b)** At rest: AES-256

### 8.2 Access Controls

**(a)** Multi-factor authentication (MFA) for all admin accounts  
**(b)** Role-based access control (RBAC)  
**(c)** Least privilege access

### 8.3 Infrastructure Security

**(a)** Firewalls and network segmentation  
**(b)** DDoS protection  
**(c)** Intrusion detection and prevention systems (IDS/IPS)

### 8.4 Monitoring and Logging

**(a)** Centralized logging  
**(b)** Audit trails for sensitive operations  
**(c)** Real-time alerting for security incidents

### 8.5 Backup and Recovery

**(a)** Encrypted backups  
**(b)** Redundant infrastructure across multiple availability zones  
**(c)** Recovery Time Objective (RTO): 4 hours  
**(d)** Recovery Point Objective (RPO): 24 hours

**Must align with SLA Schedule 1, Section 2.2.**

### 8.6 Development Security

**(a)** Secure software development lifecycle (SDLC)  
**(b)** Code review for all changes  
**(c)** Regular penetration testing (annually or after major releases)  
**(d)** Dependency scanning and vulnerability management

### 8.7 Security Audits

**Frequency:** Annual security audits and reviews
